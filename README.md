# 과학 도구상자

학생과 교사를 위한 과학 학습 도구 모음 사이트의 골격입니다.
도구 추가는 **`tools.json`에 한 칸만 추가**하면 끝입니다.

---

## 1. 파일 구성

```
science-toolbox/
├── index.html       ← 홈페이지 (수정할 일 거의 없음)
├── style.css        ← 디자인 (색·폰트 바꾸려면 여기)
├── app.js           ← 카드를 자동으로 그려주는 코드
├── tools.json       ← ⭐ 도구를 추가·수정할 때 여기만 고치면 됨
├── README.md        ← 이 파일
│
├── physics/         ← 물리 도구
├── chemistry/       ← 화학 도구
├── biology/         ← 생명과학 도구
├── earth/           ← 지구과학 도구
├── maker/           ← 메이커(만들기 활동)
└── common/          ← 공통 도구
```

각 카테고리 폴더 안에 도구별 하위 폴더를 만들고 그 안에 `index.html`을 두면 됩니다.
예: pH 계산기 → `chemistry/ph-calculator/index.html`

## 2. 주요 기능

| 기능 | 설명 |
|------|------|
| 🔍 **검색** | 이름·설명·태그·대상·활동 유형까지 모두 검색 가능 |
| 🎓 **학년 필터** | 전체 / 초등 / 중등 / 고등 칩으로 즉시 필터링 |
| 👨‍🏫 **교사용 정보 토글** | 켜면 카드에 대상·차시·준비물이 추가로 보임 |
| ⭐ **추천 도구** | `featured: true`로 표시한 도구를 상단에 모아 강조 |
| 🚧 **준비 중 자동 표시** | 실제 폴더가 없는 도구는 자동으로 회색 처리 + "준비 중" 배지 |
| 📂 **자동 하위 분류** | `subcategory` 값을 채우는 순간 자동으로 그룹화됨 |
| 📱 **모바일 대응** | 카드가 화면 크기에 맞게 자동 배치 |

## 3. GitHub Pages에 올리기 (처음 한 번)

1. GitHub에서 새 저장소를 만듭니다 (예: `science-toolbox`).
2. 이 폴더의 파일들을 모두 업로드합니다.
3. 저장소의 **Settings → Pages**로 갑니다.
4. **Source**를 `main` 브랜치, 폴더는 `/ (root)`로 설정하고 **Save**.
5. 1~2분 뒤 `https://여러분아이디.github.io/science-toolbox/` 주소로 사이트가 뜹니다.

> 처음에는 모든 도구가 "준비 중"으로 보일 거예요. 도구 폴더를 만들어 올리면 자동으로 클릭 가능 카드로 바뀝니다.

## 4. 도구 추가하기 (가장 자주 할 일)

### 4-1. 도구 파일을 해당 카테고리 폴더에 넣기

예시) 광합성 시뮬레이터를 추가한다면:

```
biology/photosynthesis/index.html   ← 실제 도구 파일
biology/photosynthesis/style.css    (필요하면)
biology/photosynthesis/script.js    (필요하면)
```

### 4-2. `tools.json`에 한 칸 추가

```json
{
  "id": "photosynthesis",
  "name": "광합성 시뮬레이터",
  "description": "빛의 세기에 따른 산소 발생량 관찰",
  "category": "biology",
  "subcategory": "",
  "path": "biology/photosynthesis/",
  "tags": ["시뮬레이션"],
  "level": "중2",
  "featured": false,
  "activityType": "탐구활동",
  "target": "중2 과학 학생",
  "sessions": "1차시",
  "materials": ["태블릿/PC"]
}
```

새 도구를 추가할 때 가장 마지막 도구 뒤에 콤마(`,`)를 찍는 걸 잊지 마세요.

## 5. 각 필드 설명

### 필수 필드

| 필드 | 설명 | 예시 |
|------|------|------|
| `id` | 영문 고유 식별자 (하이픈으로 단어 구분) | `ph-calculator` |
| `name` | 화면에 표시될 이름 | `pH 계산기` |
| `description` | 한 줄 설명 (15~30자 권장) | `산·염기 농도로 pH 계산` |
| `category` | 카테고리 — 아래 6가지 중 하나 | `chemistry` |
| `path` | 도구 폴더 경로 (끝에 `/` 필수) | `chemistry/ph-calculator/` |

**`category`에 쓸 수 있는 값**: `physics`, `chemistry`, `biology`, `earth`, `maker`, `common`

### 선택 필드 (학생용)

| 필드 | 설명 | 예시 |
|------|------|------|
| `subcategory` | 하위 분류 (비어있어도 됨) | `"역학"` |
| `tags` | 검색·필터에 잡힐 키워드 배열 | `["계산기", "산-염기"]` |
| `level` | 대상 학년 | `"고1"`, `"중2~고1"`, `"전 학년"` |
| `featured` | 추천 도구로 표시할지 (`true` / `false`) | `true` |
| `activityType` | 활동 유형 (카드에 항상 표시됨) | `"탐구활동"` |

### 선택 필드 (교사용 정보)

이 칸들은 **"교사용 정보 보기" 토글**이 켜졌을 때만 카드에 표시됩니다.

| 필드 | 설명 | 예시 |
|------|------|------|
| `target` | 수업 대상 | `"고1 통합과학 학생"` |
| `sessions` | 차시 | `"1차시"`, `"2차시"` |
| `materials` | 준비물 (배열 형태) | `["태블릿/PC", "교과서 p.45"]` |

### 메이커 전용 (선택)

| 필드 | 설명 | 예시 |
|------|------|------|
| `difficulty` | 난이도 1~3 (★ 개수) | `2` |
| `duration` | 소요 시간 | `"약 2시간"` |

## 6. 활동 유형(`activityType`) 추천 값

일관성을 위해 다음 중에서 골라 쓰는 걸 권장합니다:

- `개념학습` — 개념 이해를 돕는 자료
- `탐구활동` — 변수 조작·관찰을 통한 탐구
- `시뮬레이션` — 가상으로 현상 재현
- `실험` — 실제 재료로 수행하는 실험
- `만들기` — 직접 제작하는 활동 (메이커)
- `계산` — 수치 계산 도구
- `시각화` — 시각 자료로 보여주는 도구
- `참조` — 주기율표·상수표 같은 참고 자료

## 7. 학년 필터 작동 방식

`level` 값에 포함된 글자로 자동 분류됩니다:

- `초`가 들어있으면 → **초등** 필터에 잡힘 (예: `초6`, `초6~중1`)
- `중`이 들어있으면 → **중등** 필터에 잡힘 (예: `중2`, `중2~고1`)
- `고`가 들어있으면 → **고등** 필터에 잡힘 (예: `고1`, `중2~고1`)
- `전 학년`이면 **모든 필터에서 표시**

즉, `중2~고1`처럼 범위로 적으면 중등·고등 두 필터 모두에 나옵니다. 굳이 한 학년으로 좁힐 필요 없어요.

## 8. "준비 중" 자동 처리

페이지가 열리면 백그라운드에서 각 도구의 폴더가 실제로 있는지 확인합니다.
폴더가 없으면 자동으로:

- 카드가 회색으로 흐려짐
- "준비 중" 배지 표시
- 클릭 비활성화

따라서 **도구 이름과 정보를 미리 `tools.json`에 등록해두고 나중에 실제 폴더를 만들어도 됩니다.** 폴더가 생기는 순간 자동으로 활성화돼요. 새로고침만 해주세요.

---

## 9. 나중에 하위 분류 만들기

지금은 비워둔 `subcategory`를 나중에 채우면 자동으로 그룹화돼요.

```json
{ "name": "진자 시뮬레이터", "category": "physics", "subcategory": "역학", ... },
{ "name": "전기회로 그리기", "category": "physics", "subcategory": "전자기", ... }
```

이러면 홈에 "물리 > 역학", "물리 > 전자기"가 자동으로 나타납니다. 파일 위치·URL·외부 링크는 그대로.

권장 하위 분류:

- **물리**: 역학 / 전자기 / 광학 / 열·파동 / 현대물리
- **화학**: 일반화학 / 무기 / 유기 / 분석
- **생명과학**: 세포 / 유전 / 인체 / 생태·진화
- **지구과학**: 지질 / 대기·기상 / 천문 / 해양
- **메이커**: 전자공작 / 화학실험 / 만들기 / 코딩 등

## 10. 로컬 테스트

브라우저 보안 때문에 `index.html`을 더블클릭하면 작동하지 않습니다. 셋 중 하나를 선택:

**옵션 A — VS Code + Live Server 확장 (가장 쉬움)**
VS Code 설치 → Live Server 확장 설치 → index.html 우클릭 → "Open with Live Server"

**옵션 B — Python 한 줄**
```
python3 -m http.server 8000
```
→ 브라우저에서 `http://localhost:8000` 접속

**옵션 C — 그냥 GitHub Pages에 올려서 확인**
어차피 거기 올릴 거니까 가장 간단합니다.

## 11. 자주 막히는 부분

- **카드가 안 보여요** → `tools.json` 문법 오류. 쉼표·따옴표 누락이 가장 흔합니다. [JSONLint](https://jsonlint.com)에 붙여넣어 검사하세요.
- **클릭하면 404가 떠요** → 실제 폴더가 없는 거예요. 이젠 자동으로 "준비 중"으로 표시되니 새로고침 해보세요.
- **계속 "준비 중"으로 떠요** → `path`의 폴더 이름과 실제 폴더 이름이 다릅니다. 대소문자, 끝의 슬래시(`/`)까지 정확히 일치해야 합니다.
- **카테고리가 안 보여요** → 그 카테고리에 도구가 하나도 없으면 섹션이 안 보입니다. 도구가 추가되면 자동으로 나타납니다.
- **추가했는데 변화가 없어요** → 브라우저 캐시 때문. **Ctrl+Shift+R** (Mac: Cmd+Shift+R)로 강제 새로고침.

## 12. 디자인 바꾸기

- 카테고리 색상 → `app.js` 위쪽의 `CATEGORIES` 안에서 `color` 값 수정
- 사이트 이름·소개 문구 → `index.html`의 `<h1>`, `<p class="site-tagline">`
- 배경·폰트·간격 → `style.css`의 `:root` 부분에 변수로 모여있음

직접 고치기 어려우면 ChatGPT나 클로드에게 파일을 보여주면서 "이 부분을 이렇게 바꿔줘"라고 물어보면 됩니다.
